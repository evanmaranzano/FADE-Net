#1 Critical build_training_metadata 在 get_dataloaders 之前调用，split_fingerprint 永远为 None train.py:356 ▶
Description
build_training_metadata(cfg, seed) 在第 356 行调用，此时 get_dataloaders(cfg) 尚未执行，config.split_metadata 未填充。结果 checkpoint 元数据中 split_fingerprint 和 dataset_fingerprint 永远为 None。

#2 Critical L1 loss 与 asymmetric ordinal loss 双重惩罚，梯度偏向高估 utils.py:533 ▶
Description
当 use_asymmetric_ordinal=True 时，总损失同时包含 lambda_l1 * l1（标准 L1 回归）和 lambda_asym * loss_asym（方向加权 smooth L1）。Asymmetric ordinal loss 本应替代L1，而非叠加。
Failure Scenario
lambda_l1=0.1, lambda_asym=1.0, under_weight=2.0：5 年低估误差被 0.1×5 + 1.0×(2.0×smooth_L1(5)) 双重惩罚。梯度分布偏向高估方向，MAE 退化。
#3 Critical EMA shadow 在 backbone 解冻后仍持有初始权重，验证 MAE 使用损坏权重 utils.py:120-128 ▶
Description
EMAModel.register() 在所有参数 requires_grad=True 时注册 shadow。冻结 backbone 后 ema.update() 跳过 backbone 参数。解冻后 backbone shadow 仍为近初始权重。
Failure Scenario
freeze_backbone_epochs=10：解冻后首个 EMA 更新为 0.001×current + 0.999×stale_initial。EMA 验证使用近随机 backbone 权重数个 epoch。若此窗口内触发 "best model" 保存，选中的是损坏 checkpoint。

#11 Major LDS 权重归一化包含空 bin，稀释稀缺样本权重 dataset.py:318 ▶
Description
np.mean(weights) 包含空 bin（1/(0+1e-5) clip 到 10.0）。稀疏年龄分布下空 bin 权重 10.0 拉高均值，有效 bin 归一化后权重偏低。
Failure Scenario
